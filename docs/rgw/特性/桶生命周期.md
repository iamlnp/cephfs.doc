S3 生命周期策略是一组用 XML 编写的规则，主要用于自动管理对象的“一生”，它的核心能力有两个：
1. **过期删除 (Expiration)**：这是最常用的功能。设置一个时间，让 S3 自动删除对象。比如，自动删除 90 天前的旧日志文件，或者自动清理未完成的分段上传，释放存储空间。
2. **存储层转换 (Transition)**：为了让数据存储更经济，你可以设置规则，让 S3 把那些不经常访问的“冷数据”自动移动到更便宜的存储类别（如从标准存储转到低频访问或冷存储）。
    

> **注意**：规则中的 `Days` 是从对象的**创建日期**开始计算的[^1]。对于开启了**版本控制**的存储桶，你还可以单独管理非当前版本的过期时间，例如“保留最新版本，30天后自动删除旧版本”[^2]。

[^1]: https://help.switch.ch/engines/documentation/object-storage/s3-api-documentation/s3-bucket-lifecycle-configuration/
[^2]: https://upcloud.com/docs/guides/configure-lifecycle-policies/
