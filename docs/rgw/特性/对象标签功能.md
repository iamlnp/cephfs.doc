RGW的标签功能是一项为对象添加键值对元数据的功能，主要用于对象分类、管理和自动化操作。它于Ceph 12.2.0 (Luminous)版本引入，与AWS S3的对象标签功能兼容[^1]。

# 1 核心特性

| 特性          | 说明                                          |
| ----------- | ------------------------------------------- |
| **存储位置**    | 作为扩展属性（xattr）存储在RADOS对象的 `user.rgw.tags` 键下 |
| **数量限制**    | 每个对象最多10个标签                                 |
| **Key长度**   | 最大128字符                                     |
| **Value长度** | 最大256字符                                     |

与其他 S3 元数据的区别 ：
  
| 特性     | Tagging（标签）           | Metadata（元数据）         | ACL（权限） |
|--------|-----------------------|-----------------------|---------|
| 用途     | 分类、成本分配               | 应用层信息（如 Content-Type） | 访问控制    |
| 大小限制   | 10 个标签                | 总大小 2KB               | 不适用     |
| 修改方式   | settagging/deltagging | modify 命令             | setacl  |
| 成本报表   | ✅ 支持分组                | ❌ 不支持                 | ❌ 不支持   |
| 生命周期条件 | ✅ 可作为筛选条件             | ❌ 不可用                 | ❌ 不可用   |


# 2 主要应用场景
**场景一：生命周期管理**
- 可在生命周期规则中基于标签筛选对象，自动执行过期删除或存储层级迁移
- 例如：设置规则自动删除所有 `type=temporary` 标签的对象

**场景二：多站点数据同步过滤**
- 在跨区域同步时，可按标签筛选需要同步的对象
- 例如：`radosgw-admin sync group pipe create --tags-add=color=blue,color=red` 仅同步带有指定标签的对象

**场景三：对象分类与检索**
- 通过标签对对象进行业务分类（如部门、项目、环境等）
- 方便后续基于标签进行统计、查询或批量操作
